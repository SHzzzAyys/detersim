# Code of Conduct

## Our Pledge

We pledge to make participation in DeterSim professional, respectful, and
focused on the technical work. Disagreement is welcome when it improves clarity,
determinism, test quality, or user safety.

## Expected Behavior

- Be precise and constructive.
- Critique code, claims, and evidence rather than people.
- Preserve uncertainty when evidence is incomplete.
- Respect project boundaries and documented non-claims.
- Help keep issues and pull requests reproducible.

## Unacceptable Behavior

- Harassment, insults, or personal attacks.
- Deliberately misleading claims about project capability.
- Publishing private credentials or personal information.
- Repeatedly ignoring maintainer requests about determinism, safety, or scope.

## Enforcement

Maintainers may edit, hide, or close comments and issues that violate this code.
Repeated or severe violations may result in blocked participation.

## Scope

This code applies to project repositories, issues, pull requests, discussions,
and release channels.
